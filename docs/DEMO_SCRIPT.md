# Slidelang Demo Script — 3 minutes

## Setup
- Live URL open in one window, GitHub repo in another
- Prompt copied: "5-slide intro to vector databases for backend engineers, with one chart"
- Notifications silenced; one dry run before recording

## Beats

**0:00–0:20 — The problem.** WYSIWYG = no AI. Gamma = locked-in canvas. Neither lets you edit as code.

**0:20–0:50 — Prompt → deck.** Paste prompt, click generate. While loading: "Claude produces a typed JSON DSL, not HTML — that's how we get layout consistency and editability."

**0:50–1:30 — Editing.** Click into Monaco, change one slide. Preview updates live. "Schema-aware autocomplete, inline validation, same schema Claude used to generate."

**1:30–2:10 — Present.** Full-screen. Arrow keys. Show math slide rendering, chart slide rendering. ESC.

**2:10–2:40 — Technical depth.** Cut to GitHub. `schema/deck.py` — "5 layouts, 6 block types." `validation/rules.py` — "stable error codes feed into the repair loop."

**2:40–3:00 — Close.** Back to live preview. "Slidelang. Decks as code. Repo and docs in the description."

## Definitely show
- ✅ End-to-end prompt → preview
- ✅ Monaco edit + live update
- ✅ Present mode (briefly)
- ✅ One chart, one math slide
